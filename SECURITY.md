# Security Policy

## Supported Versions

Only the latest version of Hewcode receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Email:** hewcode.team@gmail.com

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any proof-of-concept code or screenshots

## Response Timeline

- **Within 48 hours:** Initial acknowledgment of your report
- **Within 7 days:** Assessment and severity classification
- **Within 14 days:** Patch release for critical vulnerabilities
- **Within 30 days:** Patch release for non-critical vulnerabilities

## Disclosure Policy

We will coordinate disclosure with you to ensure the vulnerability is patched before public announcement. Credit will be given in the security advisory unless you prefer to remain anonymous.

## Do NOT

- Do not open public issues for security vulnerabilities
- Do not use the issue tracker for security reports
- Do not disclose vulnerabilities publicly before coordination
